# 2. Frameworks

## GOOGLE
- **1. ASK [Perguntar]:** Definir o problema/desafio e fazer as perguntas certas pra descobrir a raiz do problema.
- **2. PREPARE [Preparar]:** Geração dos dados, coleta, armazenamento, requisitos de segurança, gerenciamento dos dados
- **3. PROCCESS [Processar]:** Limpeza dos dados, pega o que é útil e completo, descarta o que é lixo. Garante integridade.
- **4. ANALYZE [Analisar]:** Analisa os dados em busca de padrões e tendências, exploração, visualização e análise.
- **5. SHARE [Compartilhar]:** Compartilha os dados com stakeholders, explicação clara através do diálogo e gráficos.
- **6. ACT [Agir]:** Definir um plano para resolver o problema.

# 3. Insights do mercado

- 60 a 80% do trabalho do analista é na fase de limpeza de dados;
- **REGRA DE OURO:** análise sem decisões = zero valor comercial. Na hora de apresentar resultados não entregue apenas gráfico. Comece com a conclusão, exemplo: taxa de abandono subiu 18%;
- **IA:** LLms assumem tarefas pesadas de limpeza e criação de gráficos, seu papel deixa de ser de executor e passa a ser o de governador. Diferencial humano passa a ser pensamento estratégico, validando se o que a IA fez tem aderência com a realidade.
- Em projetos de big data você lida com grandes volumes de dados desestruturados. A etapa de filtragem logo no começo é vital para salvar processamento e focar apenas na resposta da pergunta do negócio
- **Outliers (anomalias):** algo que parece inválido ou picos estranhos não é necessariamente "lixo". Picos podem ser indicativos de fraudes, falhas ou começo de nova tendência.

# 4. Data Ecosystem

- Ecossistemas de dados são formados por vários elementos que interagem entre si com o objetivo de produzir, gerenciar, armazenar, organizar, analisar e compartilhar dados.
- Inclui: hardware (servidores e equipamentos), software (ferramentas e plataformas), pessoas (analistas).

## 4.1. O papel do cloud
- O cloud é fundamental pois é um local virtual onde armazenamos dados na internet. O papel do analista é aproveitar o poder do ecossistema, seja cloud ou local, para fornecer insights valiosos à equipe a fim de tomar decisões inteligentes.

# 5. Data Scientist x Data Analyst x Data Analysis x Data Analytics

- **DATA SCIENTIST (CIENTISTA DE DADOS):** foco em modelar e explorar o desconhecido. Lida com incertezas e previsões, criam novas perguintas a partir de dados brutos.
- **DATA ANALYST (ANALISTA DE DADOS):** foco em gerar insights valiosos pra responder perguntas atuais e ajudar a empresa a tomar decisões rápidas e certeiras.
- **DATA ANALYSIS (ANALISE DE DADOS):** É a ação. Processo práticos (6 fases) de coleta, transformação e organização de dados para tirar conclusões e tomar decisões.
- **DATA ANALYTICS:** conceito amplo, ciência de dados como um todo. Grande guarda-chuva que engloba o ato de analisar, o ecossistema, o gerenciamento e todas as ferramentas e métodos que os profissionais usam.

# 6. Como os dados orientam melhores decisões

- **Data-driven decision-making:** usar dados para guiar a estratégia de negócios de uma empresa, em vez de depender de achismos. Usamos dados para conhecer clientes, melhorar processos e aumentar eficiência.
- Tudo começa com o problema, a necessidade do negócio. Exemplos:
    - Melhorar reconhecimento de marca para competir com gigantes;
    - Encontrar fornecedores mais sustentáveis e éticos;
    - Entender, resolver a insatisfação e a rotatividade do RH;
    - Problema bem definido -> busca de padrões e tendências -> ação;
    - Como consequência otimiza o que já funciona ou fazer a empresa mudar drasticamente de direção.

- **SERVIÇOS DE STREAMING:** coletam dados sobre o que você assiste, analisa, usa os insights pra sugerir novos conteúdos de alta precisão. Retendo o cliente e gerando mais receitas.
- **E-COMMERCE:** dados provaram que as preferências estavam mudando das lojas físicas para o digital. Empresas que agiram baseado nisso criaram novos impérios de entrega a domicílio.
- **TELEFONIA:** dados comprovaram mudança de comportamento que fez a indústria abandonar telefones fixos. Foco nos celulares.

# 7. Fator humano

- Dados sozinhos < Dados + Experiência humana;
- Os especialistas no assunto (subject matter experts - SMEs) são as pessoas na linha de frente do negócio. O papel deles é:
    - Identificar inconsistências práticas na sua análise;
    - Dar sentido às áreas cinzentas (números não contam a história toda);
    - Validar, em última instância, as escolhas e caminhos a serem seguidos.

# Dados e Gut Instinct
- Analistas devem usar a tomada de decisão baseada em dados, através das 6 etapas:
    - **1. agir:** fazer perguntas e definir problema;
    - **2. preparar:** preparar como os dados serão coletados e armazenar as informações dos dados coletados;
    - **3. processar:** limpar os dados brutos extraíndo só o que é útil; 
    - **4. analisar:** encontrar padrões, relações e tendências;
    - **5. compartilhar:** compartilhe as descobertas com seu público restrito;
    - **6. agir:** aja com base nos dados e use os resultados.

- Outros fatores influenciam as decisões como o instinto (um "palpite"). Esse palpite se adquire com experiências anteriores de problemas já resolvidos. Pode ser também um palpite sem embasamento lógico no momento.
- Tomar cuidado pois o Gut Instinct pode ser um problema se o analista focar nele em vez de nos dados.
- Dados + conhecimento do negócio = mistério resolvido
- Misturar dados com regras de negócio é parte da rotina do analista. Para fazer a mistura certa faça as perguntas:
    - Que resultados são necessários?
    - Quem tomará as decisões apresentadas?
    - Estou respondendo à pergunta que me pediram pra responder?
    - Quão rápido essa decisão precisa ser tomada?
- Equilíbrio: um projeto urgente pode ser necessário confiar um pouco mais na experiência e conhecimento de negócios para preencher as lacunas dos dados que não deu tempo processar. Mas se o projeto oferecer tempo e recurso, foque 100% nos dados.

Pergunta Aberta (Discursiva): Na seção sobre o Fator Humano, aprendemos que Dados sozinhos < Dados + Experiência humana, e o texto destaca a importância dos SMEs (Subject Matter Experts / Especialistas no Assunto). Mesmo tendo acesso ao melhor banco de dados do mundo com um código perfeito que te dá a resposta exata, por que é vital que o Analista valide a resposta final com um SME antes de agir?
**R:** Pois o SME é o cara que está profundamento ligado ao negócio, devido ao tempo de experiência na empresa ele possui profundo conhecimento dos mínimos detalhes do negócio. Isso filtra possíveis ambiguidades e mal entendidos, como picos em determinado dia ou dados que parecem incorretos mas são válidos. Exemplo: no dia X teve menos marcações de ponto por causa do jogo do brasil. Mas na verdade o SME disse que é por conta de que os caminhões de transporte quebraram e grande parte dos funcionários foram dispensados

### Respostas Corretas e Comentários:
- **1. Letra B** - Perfeito. Você cravou a diferença exata. Cientista = prever o futuro e achar problemas novos. Analista = resolver problemas atuais de negócio para tomada rápida de decisões.
- **2. Letra D** - Exato. Muitas pessoas esquecem o terceiro pilar do ecossistema de dados. Você pode ter o melhor servidor (Hardware) e o melhor banco de dados (Software), mas se não tiver Pessoas (Analistas) para fazer as perguntas certas, o ecossistema é inútil.
- **3. Letra C** - Muito bem. Se a empresa precisa de uma decisão emergencial que afeta o fechamento do mês amanhã, o tempo de coleta, processamento e limpeza dos dados brutos não jogará a seu favor. Nesses casos raros, você precisará preencher as "lacunas" da tabela confiando na experiência humana de negócios.
- **4. Resposta Aberta** - Brilhante analogia! O seu exemplo resumiu o Fator Humano perfeitamente. O gráfico vai mostrar apenas "Faltas dispararam no dia X". A inteligência artificial ou um analista trancado em uma sala vão deduzir uma correlação estatística óbvia e errada (o jogo do Brasil). Mas o SME (Especialista), que vive a operação no chão de fábrica, traz o contexto real (caminhões quebraram). Os números são "frios" e não contam a história completa; o SME traz a "cor" para a análise.
