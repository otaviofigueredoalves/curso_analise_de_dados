## O Processo de Análise de Dados (Frameworks)
É hora de entender como os profissionais do setor passam dos dados para a decisão. Embora não haja uma única estrutura definida uniformemente por todos os analistas, há fundamentos compartilhados. Esta leitura fornece uma visão geral de vários processos.

### 1. O Processo do Certificado Google Data Analytics
Este é o processo no qual este curso se baseia e será valioso para sua carreira:
1. **Perguntar:** Definir desafio, objetivo ou questão comercial.
2. **Preparar:** Geração, coleta, armazenamento e gerenciamento de dados.
3. **Processar:** Limpeza e integridade dos dados.
4. **Analisar:** Exploração, visualização e análise.
5. **Compartilhar:** Comunicar e interpretar resultados.
6. **Agir:** Colocar os insights em prática para resolver o problema.

### 2. Processo de Análise de Dados da EMC (agora Dell EMC)
Criado por David Dietrich, este modelo reflete a natureza **cíclica** dos projetos comerciais. As fases não são marcos estáticos; elas se conectam e se repetem:
1. Descoberta
2. Pré-processamento de dados
3. Planejamento de modelos
4. Construção do modelo
5. Comunicação dos resultados
6. Operacionalização

### 3. O Processo Iterativo do SAS
Criado pela SAS (fornecedora líder de soluções de análise), este processo foca na produção de resultados repetíveis e preditivos. Ele é visualizado como um **símbolo do infinito**:
1. Perguntar
2. Preparar
3. Explorar
4. Modelar
5. Implementar
6. Agir
7. **Avaliar:** Uma etapa extra após a ação para ajudar a avaliar soluções e, potencialmente, retornar à fase de pergunta.

### 4. Processo Baseado em Projetos (Vignesh Prajapati)
Focado em cinco etapas diretas:
1. Identificação do problema
2. Design de requisitos de dados
3. Pré-processamento de dados
4. Realização da análise de dados
5. Visualização de dados

### 5. Processo de Análise de Big Data (Thomas Erl, Wajid Khattak e Paul Buhler)
Este modelo foca em dividir as etapas de "preparação e processamento" em tarefas mais granulares (específicas para o ambiente de Big Data):
1. Avaliação dos Casos de Negócios
2. Identificação de dados
3. Aquisição e filtragem de dados
4. Extração de dados
5. Validação e limpeza de dados
6. Agregação e representação de dados
7. Análise de dados
8. Visualização de dados
9. Utilização dos resultados da análise

### Principais Conclusões
Desde as pirâmides no antigo Egito até hoje, a análise evoluiu. O Processo de Análise de Dados é como a arquitetura: há diferentes maneiras de fazer as coisas, mas **as mesmas ideias centrais aparecem em cada modelo**. Quer você use a estrutura do Google ou outras, a abordagem focada e em etapas será sempre eficaz.

## Bônus: Insights Extras de Mercado (Para a sua carreira)

Os modelos 4 e 5 acima baseiam-se em artigos renomados da indústria. Abaixo, os principais "segredos de ouro" extraídos desses textos originais para você se destacar como analista:

**1. Onde você vai gastar seu tempo:** 
A indústria sabe que entre **60% a 80% do tempo de um projeto de dados é gasto na Fase de Processamento (limpando dados)** (removendo duplicatas, lidando com valores ausentes e outliers). Não negligencie essa etapa, ela é a mais crítica para que seu modelo não entregue conclusões erradas.

**2. A Regra de Ouro da Visualização:** 
Uma análise que não gera decisões tem zero valor comercial. Na hora de apresentar resultados (Fase 6), não entregue apenas um gráfico. **Comece com a conclusão** (Ex: *"A taxa de abandono subiu 18%"*) em vez de forçar as pessoas a interpretarem a imagem.

**3. O Impacto da IA na sua profissão:** 
Com a Inteligência Artificial (LLMs) assumindo tarefas pesadas de limpeza e criação de gráficos (ex: *Power BI Copilot*), o seu papel deixa de ser de "executor braçal" e passa a ser o de **"governador"**. O seu diferencial humano passa a ser o pensamento estratégico (Fase 1: Definir o problema de negócio corretamente) e validar se o que a IA fez tem aderência com a realidade da empresa.

**4. O Filtro em Projetos de Big Data:** 
Quando você for lidar com volumes massivos de dados (Big Data), você coletará muitos dados desestruturados ou cheios de "ruído". A etapa de **Filtragem** logo no começo é vital para salvar processamento e focar apenas no que tem chance de responder a pergunta de negócio.

**5. O Segredo dos Outliers (Anomalias):** 
Em grandes volumes de dados, algo que parece inválido ou com picos muito estranhos não é necessariamente "lixo" que deve ser apagado. Um pico repentino pode ser indicativo de fraude financeira, falha em um equipamento ou o começo de uma nova tendência de mercado!
