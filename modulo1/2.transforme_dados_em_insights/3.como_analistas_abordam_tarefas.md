# Como os Analistas de Dados Abordam as Tarefas

O Processo de Análise de Dados é composto por seis fases: **perguntar, preparar, processar, analisar, compartilhar e agir**. Seu objetivo é obter insights que impulsionem a tomada de decisões informadas. 

Nesta leitura, você se concentrará nas fases em si e explorará como um analista de dados pode usar esse processo para ajudar uma empresa fictícia a tomar decisões baseadas em dados sobre investimentos em treinamento.

---

## As Seis Fases da Análise de Dados
O processo ajuda os analistas de dados a decompor problemas de negócios em uma série de tarefas gerenciáveis:

1. **Perguntar (Ask):** Você trabalhará para entender o desafio a ser resolvido ou a pergunta a ser respondida. É provável que as partes interessadas lhe atribuam essa tarefa, e você fará muitas perguntas para guiar o caminho.
2. **Preparar (Prepare):** Você encontrará e coletará os dados necessários para responder às suas perguntas. Você identificará fontes, coletará os dados e verificará se são precisos e úteis.
3. **Processar (Process):** É quando você limpará e organizará seus dados (remoção de inconsistências, preenchimento de valores ausentes e formatação). Essencialmente, é garantir que os dados estejam prontos antes de iniciar a análise.
4. **Analisar (Analyze):** Fazer a análise necessária para descobrir respostas e soluções (ex: cálculo de médias, contagem de itens). O objetivo é examinar tendências e padrões.
5. **Compartilhar (Share):** Apresentar suas descobertas aos tomadores de decisão por meio de um relatório, apresentação ou visualização de dados (ex: gráficos no Google Sheets ou Tableau).
6. **Agir (Act):** Você e outras pessoas da empresa colocam os insights em ação (implementar uma nova estratégia, fazer alterações, etc.) para resolver o problema inicial.

## Colocando o Processo em Prática: Estudo de Caso

### O Dilema da Contribuição para a Aposentadoria
A equipe de gerência da Geo-Flow, Inc. percebeu que a participação dos funcionários no programa de contribuição para a aposentadoria estava abaixo do esperado, apesar do alto investimento da empresa no benefício. 

**A dúvida:** A empresa deveria criar um programa educacional para explicar os benefícios e incentivar a participação? Eles pediram uma recomendação baseada em dados.

### Como a estrutura de 6 fases foi aplicada:
1. **Perguntar:** "Os funcionários estão investindo no programa?" e "Devemos criar um programa educacional para incentivar a participação?".
2. **Preparar:** Reuniram dados do RH (dados demográficos, salários, contribuições atuais).
3. **Processar:** Limparam e organizaram os dados (removeram duplicatas/ex-funcionários e classificaram por idade, departamento e tempo de casa).
4. **Analisar:** A análise mostrou que grupos específicos de funcionários eram menos propensos a contribuir ou a saber que a empresa oferecia contribuição equivalente. Eles cruzaram dados usando visualizações para explorar diferentes contextos.
5. **Compartilhar:** Compartilharam as descobertas com a gerência usando gráficos de barras e de pizza, ilustrando que a baixa participação se concentrava em grupos específicos por falta de conhecimento.
6. **Agir:** A empresa criou um programa educacional direcionado *especificamente* para os grupos de baixa contribuição identificados. Meses depois, houve um aumento significativo nas contribuições desses grupos-alvo.

## Iteração Durante o Processo
Lembre-se de que nem sempre é possível percorrer as etapas de forma **linear**. O processo foi projetado para se basear nele mesmo (o resultado de uma etapa é o insumo da próxima).

* Se na fase de *Análise* você descobrir que usou o banco de dados errado, você deve **voltar** à fase de *Preparação*.
* Se durante o *Processamento* descobrir que a pergunta original não definia bem o problema, volte à fase de *Perguntar*.

O maior erro dos analistas é procurar respostas rápidas e pular etapas. Revise seu trabalho em cada fase para garantir qualidade e promover seu crescimento contínuo.

### Principais Conclusões
As seis fases ajudam a responder a desafios comerciais reais. Iterar (voltar e repetir quando necessário) e revisar seu trabalho durante todo o processo é fundamental para obter resultados de alta qualidade.

## Prática e Fixação: Questionário

**1. A fase de "processar" no processo de análise de dados envolve limpar e organizar os dados para garantir que estejam prontos para a análise.**
* **Resposta:** Verdadeiro. A fase de "processar" envolve limpar e organizar os dados para garantir que estejam prontos para análise.

**2. Qual das seguintes fases do processo de análise de dados envolve apresentar os resultados para os tomadores de decisão?**
* **Resposta:** c) Compartilhar (Share). Essa fase é quando você apresenta os resultados para os tomadores de decisão.

**3. Explique por que é importante iterar e revisar seu trabalho durante as fases do processo de análise de dados.**
* **Resposta:** É importante iterar e revisar porque durante o processo podem ocorrer desvios/imprevistos (como usar o banco de dados incorreto ou a pergunta inicial não definir bem o problema). Revisar ajuda a corrigir a rota e garantir a qualidade da análise.

**4. Quais tipos de dados os analistas coletaram no exemplo da empresa fictícia Geo-Flow, Inc. para analisar a participação no programa de contribuição para aposentadoria?**
* **Resposta:** Dados quantitativos (como dados demográficos, níveis salariais e contribuições atuais para aposentadoria).

**5. A fase de "agir" no processo de análise de dados é quando a empresa implementa mudanças baseadas nos insights obtidos.**
* **Resposta:** Verdadeiro. A fase de "agir" é exatamente quando a empresa implementa as mudanças baseadas nos insights.
