# Novas Perspectivas de Dados

Como você tem aprendido, é possível encontrar dados em praticamente todos os lugares. Sempre que observar e avaliar algo no mundo, estará coletando e analisando dados. Sua análise o ajuda a encontrar maneiras mais fáceis de fazer as coisas, identificar padrões para economizar tempo e descobrir novas perspectivas surpreendentes que podem mudar completamente a maneira como você vivencia as coisas.

## Estudo de Caso: Análise de Dados de Pessoas (People Analytics)

Aqui está um exemplo real de como um grupo de analistas de dados usou as seis etapas do Processo de Análise de Dados para melhorar seu local de trabalho. A história envolve a **Análise de Dados de Pessoas** (também conhecida como análise de recursos humanos ou análise da força de trabalho). 

> **Análise de Dados de Pessoas** é a prática de coletar e analisar dados sobre as pessoas que compõem a força de trabalho de uma empresa a fim de obter insights para melhorar a forma como a empresa opera.

O objetivo é definir e criar um local de trabalho mais produtivo e capacitador, desbloqueando o potencial dos funcionários, motivando-os e garantindo uma cultura empresarial justa e inclusiva.

### O Problema de Negócio
Uma organização estava tendo uma alta **taxa de rotatividade** entre os novos contratados (muitos deixavam a empresa antes de 1 ano). 

**Pergunta de negócios:** Como a organização pode melhorar a taxa de retenção de novos funcionários?

---

## O Processo de 6 Etapas na Prática

### 1. Perguntar (Ask)
Os analistas definiram o escopo do projeto e o que seria um resultado bem-sucedido. Eles fizeram perguntas colaborando com líderes e gerentes:
* O que os novos funcionários precisam aprender para serem bem-sucedidos no primeiro ano?
* Já temos dados históricos de novos funcionários?
* Gerentes com maiores taxas de retenção oferecem algo extra ou único?
* Qual é a principal suspeita de causa de insatisfação?
* Qual a meta de aumento percentual na retenção para o próximo ano?

### 2. Preparar (Prepare)
O grupo construiu um cronograma (3 meses) e definiu como relataria o progresso. Decidiram coletar dados através de uma **pesquisa online com novos funcionários**.
* **Métricas:** Desenvolveram perguntas específicas sobre contratação, integração e remuneração.
* **Segurança:** Estabeleceram regras de acesso. Dados brutos seriam restritos, apenas dados agregados/resumidos seriam visualizados (ex: faixas salariais em vez de salários individuais).
* **Prevenção:** Fizeram um brainstorming sobre possíveis problemas e como evitá-los.

### 3. Processar (Process)
Os dados precisam ser tratados com ética. Para manter a confidencialidade e segurança, adotaram as seguintes medidas:
* **Consentimento:** Garantiram que os funcionários dessem consentimento e entendessem o uso dos dados.
* **Acesso:** Restringiram o acesso a um número limitado de analistas.
* **Limpeza:** Limparam os dados para garantir que estivessem completos, corretos e relevantes, agregando informações para não revelar respostas individuais.
* **Armazenamento:** Carregaram os dados brutos em um *Data Warehouse* interno para maior segurança.

### 4. Analisar (Analyze)
Os analistas descobriram que a experiência com processos internos era um indicador-chave da satisfação.
* **Descobertas:**
  * Processo de contratação longo e complicado = maior probabilidade de deixar a empresa.
  * Processo de avaliação e feedback eficiente e transparente = maior probabilidade de permanecer.
* **Transparência:** Documentaram tudo exatamente como descobriram, para manter a confiança na pesquisa.

### 5. Compartilhar (Share)
Eles compartilharam o relatório cuidadosamente:
* Compartilharam apenas com gerentes que atingiram um número mínimo de respostas na equipe.
* Apresentaram os resultados pessoalmente para garantir o entendimento completo (contexto correto).
* Pediram que os gerentes repassassem os resultados às suas próprias equipes, gerando conversas produtivas sobre engajamento.

### 6. Agir (Act)
Trabalharam com os líderes para implementar mudanças com base nos resultados:
* **Ações Tomadas:**
  * Padronizar o processo de contratação e avaliação com base nas práticas mais eficientes e transparentes descobertas.
  * Realizar a pesquisa anualmente para comparar resultados.
* **Resultado final:** Um ano depois, a comparação indicou que a taxa de retenção melhorou! As ações foram bem-sucedidas.

## Conclusão: Análise de Dados de Pessoas é para você?
Problemas são sempre diferentes, as soluções precisam de criatividade e o impacto pode ser enorme (até mudar vidas). Como analista de dados, você pode ajudar empresas a criarem ambientes de trabalho incríveis.

### Recursos Adicionais

* **[4 Examples of Business Analytics in Action (Harvard Business School)](#):** Artigo sobre como empresas usam insights de dados para otimizar a tomada de decisões. Abaixo, os principais pontos extraídos:

  **1. Os 4 Tipos de Análise de Dados**
  * **Descritiva:** Interpretar dados históricos para identificar tendências e padrões.
  * **Diagnóstica:** Investigar e identificar a causa raiz de um problema.
  * **Preditiva:** Usar o histórico para prever resultados futuros.
  * **Prescritiva:** Usar testes (como testes A/B) para determinar qual ação trará o melhor resultado.

  **2. Analista de Negócios vs. Cientista de Dados**
  * **Analista de Negócios:** Extrai insights significativos para guiar decisões organizacionais (orçamentos, previsões).
  * **Cientista de Dados:** Transforma dados brutos em conclusões usando algoritmos complexos, programação e modelagem estatística.

  **3. Exemplos Práticos**
  * **Microsoft:** Analisaram calendários para provar que juntar a equipe reduziu o tempo de viagem para reuniões em 46%.
  * **Uber:** Usaram testes A/B para provar que a nova versão de uma ferramenta de atendimento reduziu o tempo de resolução em 7%.
  * **Blue Apron:** Previsão de demanda de ingredientes por meio de dados do cliente, evitando desperdício de comida.
  * **PepsiCo:** Segmentaram 24 milhões de lares exatos para um novo produto, guiando 80% do crescimento de vendas no primeiro ano.

  **4. A Importância da Influência**
  Como diz o professor Jan Hammond da Harvard: *"Se você for a uma reunião e outras pessoas tiverem opiniões, mas você tiver **dados para apoiar seus argumentos** e suas recomendações, **você será influente**."*
